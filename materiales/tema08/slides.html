¿Qué es Quarto?

Sistema de publicación científica y técnica de próxima generación

  • Combina texto, código y resultados en un solo documento
  • Genera documentos en múltiples formatos (HTML, PDF, Word, diapositivas)
  • Reproducible - mismo código, mismos resultados
  • Sucesor de R Markdown con mejoras significativas
  • Agnóstico de lenguaje (R, Python, Julia, etc.)

¿Por Qué Usar Quarto?

  • Reproducibilidad: Otros pueden replicar su análisis
  • Documentación: El código está explicado
  • Comunicación: Comparte resultados profesionalmente
  • Eficiencia: Actualiza datos, actualiza reporte automáticamente

Crear Documento Quarto

En RStudio:

  1. File > New File > Quarto Document...
  2. Completa título y autor
  3. Elige formato de salida (HTML, PDF, Word)
  4. Haz clic en “Create”

¿Qué es Markdown?

Lenguaje de marcado simple para estructurar documentos

  • Usa símbolos intuitivos para formato
  • Legible en texto plano (máquinas y humanos)
  • Base de los archivos .qmd
  • Similar a HTML o LaTeX, pero más simple

Header YAML (configuración) + Markdown (texto) + Code chunks (código)

title: "Mi Análisis"
author: "Tu Nombre"
format: html
---

## Sección 1

Este es texto normal en Markdown.

## Sección 2

`​`{r}
# Este es un código chunk de R
resumen <- summary(datos)
resumen
`​`

YAML Header

Header YAML: Configuración global del documento

---
title: "Mi Análisis de Datos"
author: "Juan Pérez"
date: today
format: 
  html:
    toc: true
    code-fold: true
lang: es
---
  • title, author, date: Metadatos
  • format: Tipo de salida (html, pdf, docx, revealjs)
  • toc: Tabla de contenidos
  • code-fold: Oculta código por defecto
  • lang: Idioma del documento

Code Chunks: ¿Qué son?

Chunks = Bloques de código que se ejecutan y procesan

​{r} # Este es un chunk de código R x <- 1 + 3 print(x) ​

  • Se delimitcan con ``` en la parte superior e inferior
  • Especificar lenguaje: {r}, {python}, etc.
  • Se ejecutan cuando renderizas el documento
  • Los resultados aparecen en el documento final

Chunks de Código R

```{r}
# Código R aquí
library(ggplot2)
ggplot(iris, aes(x = Sepal.Length)) +
  geom_histogram()
```
  • El código se ejecuta
  • Los gráficos se incluyen automáticamente
  • Perfecto para reproducibilidad

Opciones de Chunk

Controlan qué se muestra en el documento final

```{r}
#| label: analisis-ventas
#| echo: false
#| warning: false
#| fig-cap: "Distribución de ventas por trimestre"

# Código aquí...
ggplot(datos, aes(x = trimestre, y = ventas)) +
  geom_col()
```
  • label: Nombre del chunk (para referencias)
  • echo: false: Oculta el código, muestra solo resultados
  • eval: false: Muestra código pero no lo ejecuta
  • warning: false: Oculta advertencias
  • message: false: Oculta mensajes
  • fig-cap: Añade título a la figura

Documentos Automatizados

La potencia de Quarto: Combina todo en un documento actualizable

  • Cambio de datos? → Renderiza de nuevo y el reporte se actualiza automáticamente
  • Cambio de análisis? → El código cambia, los gráficos se regeneran
  • Reproducibilidad: Otros pueden ejecutar el mismo código con los mismos datos
  • Documentación: El código está junto con la explicación
  • Sin copia-pega: Los números en el texto se actualizan automáticamente
  • Una sola fuente de verdad: Datos, análisis y comunicación en un lugar

Ejemplo: Reporte mensual de ventas que se genera automáticamente cada mes

Renderizar Documento

Opción 1: Click en botón “Render” en RStudio/Positron

Opción 2: En consola:

quarto::quarto_render("mi_documento.qmd")

Opción 3: En terminal:

quarto render mi_documento.qmd

Formatos de Salida

Quarto puede generar múltiples formatos desde el mismo archivo

format: html         # Página web
format: pdf          # Documento PDF
format: docx         # Word
format: revealjs     # Presentación (como esta!)
format: pptx         # PowerPoint
format: typst        # Documento Typst

O múltiples formatos a la vez:

format:
  html: default
  pdf: default
  docx: default

  • ✓ Documentar tu análisis junto con el código
  • ✓ Compartir resultados de forma profesional
